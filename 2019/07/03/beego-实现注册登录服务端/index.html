beego 路由部分理解

使用 bee api pro_name 创建项目的时候会带一个 user 和 object 的模板,同时我们会看到路由内有这样一段代码

大体意思就是创建 v1 版本的 user 路由,这个地方需要注意的是beego创建的接口是严格按照 RESTFUL API 规范来的。

当我们使用 GET 方法访问/v1/user 的时候服务器对应的相应访问是 UserController 下的 GetAll

对应 GET /v1/user/1 访问的是 UserController 下的 GetOne, 而 1就是方法内的 id

这部分的路由是在 routers下的 commentsRouter_controllers.go文件中,也就是文档中描述的 beego 自动会进行源码分析，注意只会在 dev 模式下进行生成，生成的路由放在 “/routers/commentsRouter.go” 文件中。
